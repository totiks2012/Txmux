# Txmux


# TXMUX: Terminal Context Multiplexer (v3.5)

**TXMUX** — это система оркестрации терминальных вкладок. В отличие от классических мультиплексоров, которые создают виртуальные сессии внутри одного окна, TXMUX работает напрямую с вашим графическим эмулятором терминала (например, **xfce4-terminal**), управляя его нативными вкладками.

Программа позволяет мгновенно сохранять и восстанавливать целые рабочие контексты: пути к директориям, запущенные скрипты и окружение.

---

## 🛠 Архитектура системы

Система состоит из четырех модулей, работающих в связке:

1. **`tx.sh` (Оркестратор):** Единая точка входа. Проверяет состояние системы и делегирует команды движку.
2. **`txmux_03.sh` (Движок):** Ядро программы. Отвечает за парсинг сессий, запуск терминала с нужными флагами и логику формирования вкладок.
3. **`tx-watcher.sh` (Сайдкар-монитор):** Фоновый процесс («Тень»), который живет в последней вкладке. Он непрерывно сканирует активные процессы через `/proc` и обновляет файл сессии.
4. **`tx-config.conf` (Конфигурация):** Текстовый файл, где задаются глобальные переменные (тип терминала, количество слотов, интервалы обновления).

---

## 🚀 Руководство по командам

Управление осуществляется через алиас `tx` (или прямой вызов `tx.sh`).

### 1. Интерактивное меню: `tx l`

Вызывает список всех сохраненных слотов. Для каждого слота отображается:

* **ID:** Номер вкладки.
* **Программа:** Что запущено (bash, python, nano, cyan и т.д.).
* **Путь:** В какой директории находится процесс.

### 2. Режим диапазона: `tx 4_`

Команда открывает первые **N** вкладок из вашей сохраненной сессии.

* **Как работает:** `tx 4_` откроет слоты 2, 3, 4 и 5.
* **Для чего:** Быстрое развертывание базового рабочего окружения одним нажатием.

### 3. Выборочный запуск: `tx 2_4_8`

Запуск конкретных контекстов через разделитель `_`.

* **Как работает:** Откроются только те вкладки, чьи ID указаны в команде.
* **Для чего:** Если вам не нужен весь проект, а только плеер и редактор заметок.

### 4. Сохранение состояния: `tx s`

Принудительная фиксация текущего состояния всех вкладок в файл сессии.

* *Примечание:* Обычно «Тень» делает это автоматически, но команда `s` полезна перед критической перезагрузкой.

## ⚙️ Конфигурация (`tx-config.conf`)

Программа полностью отделена от жесткого кода. Вы можете настроить её под себя:

```bash
TERMINAL_BIN="xfce4-terminal"  # Эмулятор терминала
MAX_TABS=8                     # Сколько вкладок отслеживать
START_IDX=2                    # С какой цифры начинать нумерацию
WATCHER_INTERVAL=3             # Частота обновления (сек)

```

### Совместимость терминалов

Программа может использовать любой эмулятор, поддерживающий запуск вкладок через аргументы командной строки:

* **xfce4-terminal** (Рекомендовано)
* **mate-terminal**
* **lxterminal**
* **gnome-terminal** (требует `--`)

---

## 💡 Почему это удобно?

1. **Прозрачность:** Вы всегда видите статус системы в последней вкладке `WATCHER-LOG`. Если запись в файл прошла успешно, вы видите `[OK]`.
2. **Отказоустойчивость:** Если вы закроете терминал «крестиком», TXMUX восстановит всё из файла сессии при следующем запуске.
3. **Никаких лишних слоев:** Вы работаете в родном терминале вашей ОС. Буфер обмена, горячие клавиши и шрифты остаются стандартными.
4. **Автоматизация контекста:** Программа не просто открывает пустой терминал, она выполняет `cd` в вашу рабочую папку и запускает нужный инструмент (например, ваш музыкальный плеер или редактор).

---

### Как начать работу?

1. Положите скрипты в `~/.local/bin/Txmux/`.
2. Добавьте `alias tx='~/.local/bin/Txmux/tx.sh'` в ваш `.bashrc`.
3. Введите `tx l` и начните создавать свои рабочие миры.
